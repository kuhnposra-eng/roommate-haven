Acceptance checks

Given · When · Then
US1.1
Roster validation & history

Given one housemate has no task, when the roster is submitted, then activation is blocked with a field-level prompt.

US1.2
Accessible house rules

Given a preferred language is selected, when rules are viewed, then every explanation updates in one organised location.

US1.3
Private overdue reminder

Given a chore is overdue, when an admin nudges, then an in-app activity is added without personal contact.

US1.4
Unanimous proposal

Given a proposal is submitted, when all three housemates approve, then its status changes from Pending to Active.

US2.1
Accurate expense split

Given an amount and members, when an expense is saved, then each share is calculated to two decimal places.

US2.2
Real-time settlement

Given an open balance, when it is marked settled, then the ledger and net balance update without refresh.
